# דוח מצב מלא - TrackerApp

## מה זה כרגע
זוהי אפליקציית WinForms מקומית ללימוד חזרות בסגנון spaced repetition, עם מסד נתונים SQLite מקומי, ממשק עברי RTL, עץ ספרייה תורני, כרטיסי שאלה/תשובה, מנגנון דירוג חזרות בסגנון SM-2, ייצוא להדפסה, סטטיסטיקות, וגיבוי/ייצוא.

## טכנולוגיה בפועל
- שפה: C#
- UI: WinForms
- אחסון: SQLite מקומי
- רשת: גישה ל-Sefaria דרך API ציבורי
- מבנה נתונים: טבלאות `Subjects`, `StudyItems`, `ReviewHistory`, `AppMetadata`

## מה עובד מלא

### 1. פתיחת התוכנה והמסד
- האפליקציה עולה.
- יש fallback אוטומטי לתיקיית נתונים כתיבה-בטוחה אם `LocalAppData` לא זמין.
- יש יצירת/שדרוג schema אוטומטי למסד.
- יש טיפול בשגיאות startup ו-unhandled exceptions.

### 2. ממשק כללי
- ממשק עברי RTL.
- עץ ספרייה בצד ימין.
- אזור תוכן משמאל.
- סרגל עליון קלאסי עם מראה צפוף.
- לשוניות עליונות בעברית.
- שורת formatting נוספת למראה קלאסי.

### 3. כרטיסי לימוד
- טעינת כרטיסים לפי ענף בעץ.
- כרטיס שאלה/תשובה עם כותרת, נתיב, תאריך יעד, נתוני חזרה.
- 5 כפתורי דירוג:
  - גרוע
  - חלש
  - בסדר
  - טוב
  - מצוין
- עריכת כרטיס.
- מחיקת כרטיס.
- יצירת כרטיס חדש דרך dialog.

### 4. מנגנון חזרות
- קיים מנגנון SM-2 style.
- נשמרים:
  - EaseFactor
  - IntervalDays
  - RepetitionCount
  - Lapses
  - LastRating
  - LastReviewedAt
- נרשמת היסטוריית חזרות ב-`ReviewHistory`.

### 5. ספרייה תורנית - עץ בסיס
- יש Root של `תנ"ך`.
- יש `תורה`, `נביאים`, `כתובים`.
- יש lazy loading.
- ספרי תנ"ך נטענים מ-Sefaria.
- פרקים ופסוקים נבנים דינמית.
- גימטריה עברית לפרקים/פסוקים.
- נעשה normalization אוטומטי לשמות שכבר נשמרו שגוי.

### 6. אימות בפועל שבוצע
- נפתח `תנ"ך`.
- נפתחו `תורה`, `נביאים`, `כתובים`.
- נפתחו בפועל ספרים:
  - `ויקרא`
  - `יונה`
  - `רות`
- נפתח `פרק א`.
- נפתחו פסוקים.
- נוצרו screenshots אמיתיים מתוך האפליקציה.
- נוצר דוח JSON של verification.

### 7. סטטיסטיקות
- יש מסך מצב/סטטיסטיקה.
- מוצגים:
  - סה"כ פריטים
  - לביצוע היום
  - חזרות היום
  - אחוז שימור
  - נלמד היטב
- יש bar charts.
- יש line chart.
- יש heatmap.

### 8. ייצוא/גיבוי
- גיבוי SQLite עובד.
- ייצוא CSV עובד.
- ייצוא HTML להדפסה עובד.
- חלון "הדפסה לשבת" עובד.

## מה עובד חלקית

### 1. Sefaria integration
- עובד בפועל.
- יש cache מקומי.
- יש lazy loading.
- בפועל נבדקו endpoints חיים.
- אבל:
  - השילוב עדיין ממוקד לתנ"ך בלבד.
  - המימוש כרגע לא מכסה את כל ספריית Sefaria הכללית.
  - חלק מהשירות הישן עדיין קיים בקוד במקביל לשירות החדש.

### 2. עץ הספרייה
- תנ"ך עובד בפועל עד פסוק.
- התלמוד קיים במערכת הזריעה והמיפוי.
- אבל לא בוצע באותו עומק של verification חזותי/שימושי כמו תנ"ך בסבב האחרון.

### 3. סטטיסטיקות
- הנתונים מחושבים ומצוירים.
- אבל ה-UI הוא custom painted בסיסי, לא dashboard "מוצר פרימיום" גמור.
- אין drill-down אינטראקטיבי.

### 4. CSV
- backend של import קיים.
- export מחובר ל-UI.
- import עדיין לא מחובר לכפתור/flow UI ברור במסך הראשי.

### 5. הדפסה
- HTML נקי עובד.
- PDF לא ממומש.

## מה קיים בקוד אבל לא מחובר טוב ל-UI
- `ImportFromCsv(...)` קיים ב-backend.
- אין כרגע מסלול UI מלא נוח למשתמש ל-import bulk.
- הוספת תיקיות custom קיימת ב-backend (`AddSubjectFolder`) אבל לא מחוברת למסך tree כ-feature גמור.

## בעיות ידועות
- יש קבצים שבהם צפייה דרך טרמינל/PowerShell מציגה ג'יבריש, למרות שבתוכנה עצמה זה יכול להיות תקין; זו בעיית encoding של סביבת shell ולא בהכרח של ה-UI.
- יש קוד תרגום/מיפוי ישן ומעט כפול בכמה קבצים.
- יש שירות Sefaria ישן ושירות Sefaria חדש; בפועל החדש בשימוש.
- חלק מהכותרות/strings בהדפסה עדיין באנגלית.
- ה-UI הכללי עדיין לא clone מושלם לחלוטין של Kal Lizkor; הוא קרוב בכיוון, אבל עדיין לא one-to-one.
